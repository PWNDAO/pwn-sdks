# API Reference

## Core Components

### Strategy

The core data structure representing a lending strategy.

```typescript
interface Strategy {
  id: string;
  name: string;
  description?: string;
  terms: {
    creditAssets: Asset[];
    collateralAssets: Asset[];
    apr: Record<string, number>;
    ltv: Record<string, number>;
    durationDays: number;
    expirationDays: number;
    minCreditAmountPercentage: number;
  };
  lendingStats: {
    totalCommittedAmount: bigint;
    totalUtilizedAmount: bigint;
    totalAvailableAmount: bigint;
  };
  borrowingStats: {
    totalBorrowedAmount: bigint;
    totalRepaidAmount: bigint;
    totalDefaultedAmount: bigint;
    activeBorrowedAmount: bigint;
  };
  curator?: {
    name: string;
    description?: string;
    avatar?: string;
  };
}
```

### Asset

Represents a token that can be used as credit or collateral.

```typescript
interface Asset {
  address: string;
  chainId: number;
  name: string;
  symbol: string;
  decimals: number;
}
```

### repayLoan

Core function to repay an active loan.

```typescript
async function repayLoan(
  {
    loanId: bigint,
    repayer: AddressString,
    chainId: SupportedChain,
  },
  deps: {
    loanContract: {
      repayLoan: (loanId: bigint, chainId: SupportedChain) => Promise<void>
    }
  }
): Promise<void>
```

The `repayLoan` function allows users to repay an active loan. It requires:
- `loanId`: The ID of the loan being repaid
- `repayer`: The address making the repayment (must be a valid address)
- `chainId`: The chain ID where the loan exists
- `deps.loanContract`: An instance of the loan contract with the `repayLoan` method

This function handles validation of inputs and interacts with the blockchain to process the repayment transaction.

## Types

### ProposalParamWithDeps

Type representing a proposal with its dependencies.

```typescript

  type: T;
  params: Parameters<(typeof proposalTypes)[T]>[0];

  };
```

## Error Handling

All hooks and functions may throw or return errors in the following cases:
- Network errors
- Invalid input parameters
- Unauthorized access
- Contract interaction failures
- Wallet connection issues
- Nonce management errors
- Invalid nonce revocation attempts
- Nonce space validation failures
- Invalid loan ID (for repayLoan)
- Invalid repayer address (for repayLoan)
- Attempting to repay a loan that doesn't exist or is already repaid
- Loan is not active or has already defaulted

It's recommended to always implement proper error handling and loading states in your application.
